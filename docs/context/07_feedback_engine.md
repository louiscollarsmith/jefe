# Feedback Engine

Feedback is part of the product and part of the company operating system.

## Goal

Turn merchant feedback, rejected recommendations, sales conversations, support tickets, churn reasons and feature requests into structured product signals.

## Flow

1. Capture
   - one tap in app
   - text feedback
   - voice note
   - screen recording later
   - daily brief feedback

2. Interpret
   - transcribe audio
   - summarise pain
   - identify implied request
   - extract severity
   - extract sentiment
   - attach merchant/action/context

3. Confirm
   - play distilled summary back to merchant
   - “sounds like you want X because Y — did I get that right?”
   - one tap confirm/correct

4. Route
   - create/update Linear issue
   - attach recording/transcript
   - attach merchant, plan, MRR, risk, action type

5. Order
   - dedupe similar asks
   - cluster themes
   - weight by revenue and churn risk
   - founders decide final priority

6. Close loop
   - when shipped, notify everyone who asked
   - include in changelog
   - track whether it improves retention/usage

## Guardrail

Revenue-weighted feedback informs roadmap. It does not blindly decide roadmap.
